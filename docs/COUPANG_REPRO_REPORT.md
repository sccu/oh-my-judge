# 쿠팡 안티봇 우회 및 자동화 재현 보고서 (2026-03-11)

## 1. 실험 목표
쿠팡의 강력한 안티봇 솔루션(Akamai Premier)을 우회하여, 사람의 개입 없이 여러 키워드에 대한 검색 결과를 정형 데이터(JSON)로 안정적으로 추출하는 자동화 시스템 구축.

## 2. 주요 시행착오 및 해결 과정

### 단계 1: 환경 검증(Fingerprinting)의 벽
*   **현상**: 기본 Playwright Headless 모드 사용 시 `goto()` 직후 **403 Forbidden (Access Denied)** 발생.
*   **원인**: 브라우저 지문 및 자동화 흔적이 TLS 핸드쉐이크 및 초기 로드 단계에서 즉시 탐지됨.
*   **해결**: **CDP(Chrome DevTools Protocol)** 방식을 도입. 실제 설치된 크롬 브라우저를 9222 포트로 띄우고 에이전트가 이를 제어하게 함으로써 '환경 검증'을 100% 통과함.

### 단계 2: UI 식별 및 셀렉터 불일치
*   **현상**: 메인 페이지 진입에는 성공했으나 검색창을 찾지 못함.
*   **원인**: 쿠팡의 검색창이 ID(`#`)가 아닌 클래스(`.`) 기반으로 변경되었으며, 자동화 탐지 시 DOM 구조를 미세하게 변조함.
*   **해결**: 실제 브라우저의 DOM을 정밀 분석하여 `input[name='q']`, `input[title='쿠팡 상품 검색']` 등 변하지 않는 **속성 기반 셀렉터**를 확보하여 성공률을 높임.

### 단계 3: 지연 공격(Tarpit)과 타임아웃
*   **현상**: 스크린샷 캡처 및 결과 리스트 대기 중 30초 이상의 무한 대기 및 타임아웃 발생.
*   **원인**: Akamai가 의도적으로 응답 속도를 늦춰 자동화 도구의 리소스를 고갈시키는 전략 사용.
*   **해결**: 
    - `wait_until='commit'` 설정을 통해 페이지 로딩 완판을 기다리지 않는 **'성급한 사용자'** 모드 도입.
    - 스크린샷 타임아웃을 5초로 극단적으로 줄이고 실패 시 즉시 무시하도록 설계하여 데이터 추출 흐름을 보호함.

## 3. 최종 구현된 핵심 기술 (Key Features)

1.  **멀티태스킹 웜업 (Advanced Warm-up)**:
    - 베지에 곡선을 활용한 비선형 마우스 이동.
    - 마우스 이동과 휠 스크롤을 실시간으로 병행(Interleave)하여 실제 사용자의 시선 이동 모사.
    - 검색창 클릭 후 '노트' 입력 -> 지우기 -> 1초 대기라는 정교한 인지 행동 시뮬레이션 포함.

2.  **최적화된 세션 관리**:
    - 웜업은 최초 1회만 수행하여 효율성 극대화.
    - 검색 결과 페이지(SERP)에서 바로 다음 검색을 이어가는 연속 검색 루프 구현.

3.  **기계 친화적 JSON 추출**:
    - 할인가, 할인율, 별점, 리뷰 수, 광고 여부 등 화면의 모든 정보를 `JSON_RESULT` 스트림으로 한 줄씩 출력.
    - 상품 카드의 모든 `data-*` 속성을 수집하여 향후 분석을 위한 메타데이터 확보.

## 4. 최종 확인된 사실 (Facts)
*   **Bypass Success**: CDP + 정교한 행동 패턴 조합 시 Akamai의 실시간 행동 분석을 안정적으로 통과함.
*   **Performance**: 페이지 완전 로드를 기다리지 않는 'Impatient Mode'가 안티봇 탐지 회피와 속도 측면에서 모두 유리함.
*   **Stability**: 직접 터미널에서 쉘 스크립트 `./run_repro.sh`를 통해 실행할 때 가장 부드럽고 안정적으로 동작함.

## 5. 향후 확장 및 Headless 대응 전략

### 5.1. Headless vs Headful 이슈
*   **현재 상태**: 안정적인 우회를 위해 `headless=False` (Headful) 모드와 유저 프로필 활용이 필수적임. 
*   **--headless=new 테스트 결과 (실패)**: 최신 크롬의 '새로운 헤드리스' 모드를 시도했으나, Akamai에 의해 즉시 식별되어 `Search input not found` (차단 페이지 노출) 오류 발생. 이는 환경 검증 단계에서 지문(Fingerprint) 불일치로 인해 차단되었음을 의미함.
*   **결론**: 일반적인 Headless 방식으로는 쿠팡의 보안을 통과할 수 없음.

### 5.2. 리눅스 서버(CI/CD) 환경 대응 (Xvfb 전략)
향후 본격적인 서버 구동 시, 실제 모니터 없이도 Headful의 안정성을 확보하기 위해 다음 전략을 권장함:
1.  **Xvfb (X Virtual Framebuffer)**: 리눅스 메모리 상에 가상 디스플레이를 생성.
2.  **구동 방식**: 브라우저는 `headless=False`로 실행하되, 출력 대상을 Xvfb로 지정하여 시스템 입장에서는 Headful이지만 유저 눈에는 보이지 않는 상태로 운영.
3.  **장점**: 안티봇의 '눈에 보이는 브라우저인가?' 검증을 통과하면서도 서버 환경에서 백그라운드 구동 가능.

## 6. 결론
사용자님과의 협업을 통해 **"환경 차단 우회 -> UI 인식 해결 -> 지연 공격 방어 -> 행동 정교화"**라는 고난도 안티봇 우회의 정석적인 단계를 모두 정복했습니다. 이제 이 시스템은 쿠팡의 최신 방어 체계에서도 안정적으로 데이터를 수집할 수 있는 강력한 도구가 되었습니다!
